Reactive HTML, server swaps, streams

YugaJS

A lightweight browser library for building modern, server-driven interfaces with plain HTML attributes, tiny reactive scopes, jQuery-like helpers, DOM morphing, and EventSource streams.

Install Browse Directives

Installation

Use the built file directly in the browser, or import the source entry while developing with Vite.

Browser Script

<script src="/dist/ys.min.js"></script>
<script>
    YS.start()
</script>

Vite Development

<script type="module" src="/dist/ys.esm.js"></script>
<script type="module">
    YS.start()
</script>

Local Commands

npm install
npm run dev
npm run build

Essentials

A YugaJS component starts at an element with ys-data or ys-component. Expressions run against that component scope.

<div ys-data="{ count: 0 }">
    <button @click="count++">+</button>
    <button @click="count--">-</button>
    <strong ys-text="count"></strong>
</div>

Scopes

Each ys-data block gets a reactive object. Directives inside read and write properties from that object.

Events

Use @click, @submit.prevent, and other event bindings. The handler receives $event and $el.

Updates

DOM updates are scheduled automatically. Use YS.nextTick() when you need to run code after the flush.

Expression Context

Directive expressions run with the current scope in context. Event handlers also receive $event and $el.

<button @click="count++; console.log($event.type, $el)">
    Add
</button>

Nested Scopes

Nested ys-data elements create independent scopes. Inner expressions read from their own scope first.

<div ys-data="{ count: 0 }">
    Parent: <span ys-text="count"></span>

    <div ys-data="{ count: 100 }">
        Child: <span ys-text="count"></span>
    </div>
</div>

Directives

Directives are HTML attributes that connect DOM behavior to a component scope. Short forms are supported for events and bindings.

ys-data

component scope

Creates a reactive scope for an element and its children.

<div ys-data="{ name: 'Aisha', open: true }">
    <span ys-text="name"></span>
</div>

ys-text and ys-html

output

ys-text writes text content. ys-html writes raw HTML, so only use trusted content.

<p ys-text="message"></p>
<div ys-html="trustedMarkup"></div>

ys-model

forms

Two-way binding for text inputs, textareas, selects, checkboxes, radios, file inputs, ranges, numbers, and contenteditable elements.

<input ys-model="name">
<input type="checkbox" ys-model="enabled">
<select multiple ys-model="roles">...</select>

@event / ys-on:event

events

Runs an expression when an event fires. Modifiers include prevent, stop, self, once, and debounce.

<button @click="count++">Add</button>
<form @submit.prevent="save()">...</form>
<input @input.debounce="search = $event.target.value">

:attr / ys-bind:attr

attributes

Binds any attribute to an expression. False, null, and undefined remove the attribute; true writes a boolean attribute.

<button :disabled="saving">Save</button>
<a :href="'/users/' + id">Profile</a>

ys-class

classes

Accepts a string, array, or object of class names. Object keys may contain multiple class names.

<p ys-class="{ 'active danger': selected, muted: !selected }">
    Status
</p>

ys-show and ys-if

visibility

ys-show toggles display. ys-if mounts or removes content. Add ys-transition and optional ys-duration for built-in transitions.

<div ys-show="open" ys-transition>Panel</div>
<template ys-if="users.length === 0">
    <p>No users yet</p>
</template>

ys-for and ys-key

lists

Loops over arrays with optional index. Add ys-key for stable DOM reuse and animated moves. Use <template> when the loop should render several nodes.

<li ys-for="user, index in users" ys-key="user.id">
    <span ys-text="user.name"></span>
    <button @click="users = users.filter((_, i) => i !== index)">Remove</button>
</li>

ys-init and ys-effect

lifecycle

ys-init runs once when the element compiles. ys-effect re-runs whenever dependencies used by the expression change.

<div ys-init="loaded = true" ys-effect="console.log(count)"></div>

ys-cloak

first paint

Hide uncompiled UI until YugaJS starts. Add a CSS rule for the initial page load.

<style>[ys-cloak] { display: none !important; }</style>
<div ys-data="{ ready: true }" ys-cloak>Ready</div>

ys-confirm

guard

Shows a native confirmation dialog before the click continues.

<button ys-confirm="Delete this user?" ys-post="/users/1/delete">
    Delete
</button>

ys-key

keyboard

Runs an expression when a keyboard combo is pressed. Combos can include ctrl, cmd, meta, shift, and alt. Add .window for a window-level listener.

<div ys-data="{ open: false }">
    <div
        ys-key:ctrl+k.window="open = true"
        ys-key:escape.window="open = false"></div>

    <div ys-show="open" ys-cloak>Search</div>
</div>

ys-teleport and ys-slot

composition

ys-teleport moves an element into another DOM target. ys-slot renders the original child markup inside a component template.

<div ys-teleport="#modals">Modal content</div>
<div ys-slot></div>

ys-preview and ys-dropzone

uploads

ys-preview renders selected files into a target. ys-dropzone connects a drop area to a file input and adds ys-dragover while files hover over it.

<input id="avatar" type="file" ys-preview="#preview">
<div ys-dropzone="#avatar">Drop files here</div>
<div id="preview"></div>

ys-memo

performance

Skips recompiling a subtree while the memo expression is unchanged.

<div ys-memo="user.id">
    Expensive user fragment
</div>

Server Requests

YugaJS can submit forms, send GET/POST/PUT/PATCH/DELETE requests, swap HTML, morph DOM, update state from JSON, show loading states, cache GET responses, and follow redirects.

ys-get

Runs on click by default or on form submit. Use ys-trigger for DOM events, load, polling, intersection, modifiers, or multiple triggers.

Mutation directives

ys-post, ys-put, ys-patch, and ys-delete submit forms as FormData, or send JSON for non-form elements.

<form ys-post="/users" ys-target="#users" ys-swap="append" ys-reset>
    <input name="name" placeholder="Name">
    <button ys-loading-text="Saving...">Save</button>
</form>

<div id="users"></div>

PUT, PATCH, DELETE, and Request Bodies

Non-form mutations send the current scope as { state: ... }. Use ys-send for a specific evaluated payload, or ys-no-body to omit the request body.

<button ys-patch="/users/15" ys-send="{ active: false }">
    Deactivate
</button>

<button ys-delete="/users/15" ys-no-body>Delete</button>

<div ys-put="/heartbeat" ys-trigger="every 30s" ys-no-body></div>

Request Triggers

ys-trigger accepts comma-separated triggers on GET and mutation directives. Use any DOM event, load, every 500ms/every 5s, or intersect (also visible).

<input ys-get="/search" ys-trigger="input.debounce.300ms">
<div ys-get="/next-page" ys-trigger="intersect.once.threshold.0.5"></div>
<button ys-post="/save" ys-trigger="click.once, keydown.ctrl.enter">Save</button>

Event modifiers include once, prevent, stop, capture, passive, window, document, debounce, throttle, and keyboard modifiers such as enter, escape, ctrl, and shift.

Search As You Type

<div ys-data="{ selectedName: '', selectedId: '' }">
    <input
        name="search"
        ys-get="/select-users"
        ys-trigger="input"

Global Headers and CSRF

Call YS.config() before YS.start(). Global headers may be an object or a function. Enabling CSRF reads a token from a meta tag on every request.

<meta name="csrf-token" content="your-token">
<script>
YS.config({
    headers: { Accept: 'application/json' },
    csrf: true
})
YS.start()
</script>

<button ys-get="/profile" ys-headers="{ 'X-View': 'compact' }">
    Load profile
</button>

By default the token is read from meta[name="csrf-token"] and sent as X-CSRF-TOKEN. Customize this with csrf: { meta: 'csrf', header: 'X-CSRF' }.

Swap Strategies

StrategyBehaviorUse it when
htmlReplaces the target's inner HTML.The target is a container for server content.
appendInserts the response at the end of the target.Adding rows, messages, logs, or toasts.
prependInserts the response at the beginning of the target.Newest-first feeds and notifications.
replaceReplaces the target element with the response root.The server returns a complete replacement node.
outerReplaces the target, or removes it for an empty response.Deleting or replacing one item from a list.
morphMorphs the target while preserving useful client-side state.Refreshing complex UI with inputs or stateful children.

Action Requests

ys-action posts a JSON payload containing an action name and the current clean scope state. It is useful when a server endpoint behaves like a command handler.

<div ys-data="{ count: 0, message: 'Ready' }">
    <button ys-action="/counter-action" ys-send="increment">+</button>
    <button ys-action="/counter-action" ys-send="decrement">-</button>
</div>
{
    "action": "increment",
    "state": {
        "count": 0,
        "message": "Ready"
    }
}

Request Events

YS.on('request:start', ({ url, method }) => {
    console.log('Starting', method, url)
})

YS.on('request:finish', ({ url }) => {
    if (url.includes('/delete-user')) {
        document.querySelector('#toast').textContent = 'User deleted'
    }
})

Available events include request:start, request:success, request:error, request:finish, and request:validation-error.

Streaming

Use ys-stream to open an EventSource connection. Server-sent events can swap fragments, patch state, dispatch browser events, redirect, focus fields, or close the stream.

<div ys-data="{ count: 0 }">
    <div ys-stream="/stream" ys-target="#messages" ys-swap="append"></div>
    <div id="messages"></div>
</div>

Supported Event Names

EventBehavior
messageUses the element's default ys-swap, which defaults to append.
append, prepend, html, replace, morphSwaps the event data into the target with the matching strategy.
signalParses JSON and applies it to the current scope.
dispatchParses { "event": "...", "detail": {} } and dispatches a CustomEvent on window.
redirectNavigates to the event data URL.
focusFocuses the selector in the event data.
closeCloses the EventSource connection.

<div ys-stream="/chat/stream" ys-action-url="/chat/action">
    <button ys-stream-action="like">Like</button>
</div>

SSE Signals

<div ys-sse-signals="/signals"></div>

Signal events should include a target selector and a signals object, for example { "target": "#counter", "signals": { "count": 2 } }.

Stream Events

Listen with YS.on() for stream:start, stream:message, stream:signal, stream:dispatch, stream:error, stream:close, stream:action:start, stream:action:success, and stream:action:error.

Components

Register named components in JavaScript, then attach them to markup with ys-component. Component data, methods, props, slots, stores, and lifecycle helpers are merged into one reactive scope.

YS.component('user-card', {
    data() {
        return { open: false }
    },
    methods: {
        toggle() {
            this.open = !this.open
        }
    },
    init() {
        console.log('mounted', this.props)
    }
})

YS.start()
<article ys-component="user-card" ys-props="{ name: 'Hamid' }">
    <button @click="toggle()">Toggle</button>
    <p ys-show="open" ys-text="props.name"></p>
</article>

Component Data Overrides

When an element has both ys-component and ys-data, the inline data is merged into the component data. This is useful for multiple instances with different initial state.

<div ys-component="counter"></div>
<div ys-component="counter" ys-data="{ count: 10 }"></div>

Component Watchers

YS.component('counter', {
    data() {
        return { count: 0 }
    },
    methods: {
        increment() {
            this.count++
        }
    },
    init() {
        this.$watch(
            function () {
                return this.count
            },
            function (value, oldValue) {
                console.log('Counter changed:', oldValue, value)
            }
        )
    }
})

Reactivity

YugaJS includes small reactive primitives for standalone state, global stores, watchers, computed values, and signals.

Stores

const auth = YS.store('auth', {
    user: null
}, { persist: true })

auth.user = { name: 'Aisha' }

Signals

YS.signalEffect(() => {
    console.log(count())
})

count.set(1)

Stores In Markup

    <button @click="$store.counter.count++">+</button>

DOM Morphing

Morphing updates a target with server-rendered HTML while preserving useful client-side state, including form values and attributes owned by reactive directives.

Request Morphing

<button ys-get="/profile" ys-target="#profile" ys-swap="morph">
    Refresh profile
</button>

<section id="profile">
    Current profile markup
</section>

Programmatic Morphing

YS.morph(
    document.querySelector('#profile'),
    '<section id="profile"><h2>Aisha</h2></section>'
)

What Is Preserved

Form State

Inputs, textareas, and selects keep their current values during morph updates.

Managed Attributes

Reactive attributes owned by directives like ys-class, ys-show, ys-bind, and ys-cloak are preserved.

Use morph for complex refreshed UI. Use html, append, or prepend for simple fragments where preserving existing DOM state is not important.

Plugins

Plugins receive the YugaJS object and can register components, directives, collection methods, events, or helper functions.

function ToastPlugin(YS, options = {}) {
    YS.toast = message => {
        const el = document.createElement('div')
        el.textContent = message
        document.body.appendChild(el)
        setTimeout(() => el.remove(), options.duration || 2400)
    }
}

YS.use(ToastPlugin, { duration: 3000 })
YS.toast('Saved')

Collection Plugins

YS.fn('highlight', function () {
    return this.each(el => {
        el.style.outline = '2px solid #137b67'
    })
})

$('.card').highlight()

Custom Directives

YS.directive('focus', {
    mounted(el) {
        el.focus()
    }

Object Plugins

const HelloPlugin = {
    name: 'hello-plugin',

    install(YS, options) {
        YS.directive('hello', {
            mounted(el) {
